# QB × Roster Interaction Overlay Experiment

## Research Question

Does position-group availability improve prediction when applied **only** on top of games where the QB overlay gate is already active (QB stability is fragile)?

## Motivation

The standalone roster overlay experiment found that position-group availability overlays applied to all games added noise, not signal.

However, the hypothesis was too broad: roster availability may matter **specifically when QB stability is already fragile**. A team with a QB change AND a depleted offensive line is worse off than a team with a QB change alone. This interaction is what we test here.

## Architecture

```
Layer 1 (fixed): QB overlay (H. changed OR starts<17, cap=40, gamma=1.0)
Layer 2 (swept):  Position-group overlay on top, only where
                  QB gate is active AND position depletion > threshold

final_logit = base_logit
            + qb_overlay * qb_gate_mask
            + roster_overlay * qb_gate_mask * roster_gate
```

This creates 4 game types:

| QB Fragile? | Roster Depleted? | Outcome |
|------------|-----------------|---------|
| No | No | Base incumbent probability |
| Yes | No | QB overlay only |
| No | Yes | Base incumbent (roster alone not applied) |
| Yes | Yes | QB overlay + roster overlay |

## Baselines

**v2.0.0 (pre-overlay base):** Standard Elo + qb_changed + rolling_mov_3 + Platt. Holdout LL: 0.6259, Val LL: 0.6341.

**v3.0.0 / L1 Frozen QB Overlay:** Same base + frozen QB overlay (gate: H. changed OR starts<17, cap=40, gamma=1.0). This is the current champion — L1 in this experiment reproduces it. Val LL: 0.6305, Holdout LL: 0.6200.

## Decision

**Note on v3.0.0 / L1:** This layer reproduces the v3.0.0 Frozen QB Overlay champion (val LL 0.6305, holdout LL 0.6200). The layered architecture confirms v3.0.0 is correct. The question below is whether roster overlays add information beyond what the QB overlay already captures.

**❌ REJECTED**

No QB × roster interaction overlay beats layer 1 (QB overlay alone) on both validation and holdout.

## Takeaways

1. **Interaction is sharper but still too weak** — even within the QB-fragile subset, position-group availability adds only noise-level signal.
2. **Availability scores remain too coarse** — simple OUT counts don't distinguish starter vs backup, snap share, or unit context.
3. **QB overlay absorbs all the signal** — the QB overlay already captures the biggest game-to-game variance factor. Position-group availability is double-counting what Elo already learns.
4. **Richer weighting needed** — player-level impact weights or snap-share adjustments may be required before position-group features become useful beyond the QB position.
